## 1. 시스템 아키텍처

본 시스템은 선박 내 의료관리자 부재 또는 원격 의료 지원 지연 상황에서 **즉각적인 응급 판단**을 돕기 위해 설계되었습니다.

```
[선원 스마트폰 촬영]
        ↓
[m_medic_v2.py 통합 시스템]
  ├─ ML 엔진: RandomForest → 해양사고 위험도 예측
  └─ DL 엔진: MobileNetV3 Large → 외상 6종 분류 + 신뢰도
        ↓
[출력]
  ├─ 위험등급 (CRITICAL / HIGH / MEDIUM / LOW)
  ├─ 응급처치 매뉴얼 (선원법 근거)
  └─ JSON 진단 리포트 저장
```

- **Input:** 외상 부위 사진 (JPG/PNG)
- **AI Core:** MobileNetV3 Large (6종 외상 분류) + RandomForest (사고 위험도)
- **Knowledge Base:** 선원법 시행규칙 [별표 5의5] + 위험도별 응급처치 지침
- **Output:** 진단명, 위험도, 즉각 조치 사항, 법적 근거, JSON 리포트

---

## 2. 통합 진단 시스템 주요 기능

### 2.1 외상 자동 식별 (DL)
- 스마트폰 사진 1장으로 외상 6종 즉시 판독 (< 200ms)
- 신뢰도와 함께 상위 3개 후보 클래스 출력
- 저신뢰도(< 50%) 시 경고 메시지 자동 출력

### 2.2 위험등급 및 응급처치 매뉴얼 자동 생성

| 위험등급 | 해당 외상 | 조치 방향 |
| :--- | :--- | :--- |
| CRITICAL | 자창(Stab_wound) | 즉시 가항 항구 회항, 지혈 + 쇼크 방지 |
| HIGH | 화상, 열창 | 항행 중 처치, 2차 감염 방지 |
| MEDIUM | 절창 | 지혈·봉합, 감염 모니터링 |
| LOW | 찰과상, 타박상 | 현장 처치 후 경과 관찰 |

### 2.3 법적 근거 자동 연계
- 진단 클래스별 선원법 시행규칙 조문 자동 출력
- 의료관리자 업무 범위 내 처치 가이드 제공

### 2.4 JSON 진단 리포트 저장
- 진단 결과를 `report_YYYYMMDD_HHMMSS.json` 형식으로 자동 저장
- 추후 원격 의료진 검토 및 의무기록 활용 가능
