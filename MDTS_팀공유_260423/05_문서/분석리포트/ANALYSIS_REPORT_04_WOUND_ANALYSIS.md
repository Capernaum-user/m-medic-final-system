# 🩹 Project Maritime-Medic (M-Medic) 분석 보고서 #04

**작성일:** 2026-04-10  
**담당자:** Gemini CLI (빅데이터 분석가)  
**주제:** 해양사고 기반 외상(Wound) 패턴 분석 및 시스템 설계

---

## 1. 해양사고-외상 상관관계 분석 (Correlation Analysis)

수집된 해양사고 통계(`marine_accidents_history.csv`)와 의료 지침을 결합 분석한 결과, 선상에서 발생하는 주요 외상 패턴은 다음과 같습니다.

| 사고 유형 | 예상 주요 외상 (Wound Types) | 응급처치 우선순위 (Priority) |
| :--- | :--- | :--- |
| **화재/폭발** | 2도 이상의 화상 (Burns), 흡입 화상 | 기도 확보 및 환부 냉각 |
| **선박 충돌** | 다발성 골절, 열상 (Lacerations), 타박상 | 지혈 및 부목 고정 |
| **작업 중 사고** | 절상 (Cuts), 절단 (Amputation), 찰과상 | 감염 방지 및 신속한 봉합 준비 |
| **익수 사고** | 저체온증, 폐렴, 피부 연조직염 | 체온 유지 및 심폐소생 |

---

## 2. 데이터셋 확보 전략 (Data Augmentation Plan)

현재 외상 이미지의 클래스별 균형을 맞추기 위해 다음과 같은 전략을 활용하고 있습니다.
- **오픈소스 데이터 활용:** Kaggle `wcs_wound` 및 `yasin_wound` 데이터셋 병합.
- **증강 데이터(Augmentation):** 부족한 클래스(자창 등)에 대해 RandomRotation, ColorJitter, 가우시안 노이즈 등 기법 적용.

---

## 3. 시스템 설계 제안 (System Architecture)

### 3.1 실시간 진단 엔진
- 사용자가 부상 부위 사진 촬영 -> AI 모델(ResNet)이 외상 종류 판별 -> **선원법 의료관리자 가이드라인** 자동 매칭.

### 3.2 선내 비상 대응 모듈
- 진단된 부상 정보가 즉시 선내 의료 관리 기록(표준의료보고서)으로 자동 전산화되는 기능 통합.

---

## 4. 관리자 메모
- 외상 이미지 데이터의 다양성 확보를 위해 추가 소스를 지속적으로 탐색 중.
- 분석 파이프라인은 `wound_pipeline.py`에 구현되어 데이터가 확보되는 즉시 가동 대기 상태임.
