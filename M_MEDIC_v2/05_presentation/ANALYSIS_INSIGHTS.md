## 1. ML 결과에서 얻을 수 있는 정보

### 1.1 머신러닝 (해양사고 통계 분석) - 핵심 인사이트

#### Feature Importance 분석 결과가 말해주는 것

| 순위 | 변수 | 중요도 | 실무 해석 |
|------|------|--------|-----------|
| 1 | 사고유형 | ~0.38 | **화재·침몰 사고가 사상자 발생의 최대 예측 인자** → 화재 감지 센서 투자 우선 |
| 2 | 선박종류 | ~0.28 | **어선이 사상자 위험의 핵심** → 어선 집중 안전 교육 정책 필요 |
| 3 | 사고원인 | ~0.18 | **운항과실(38%) 차지** → 피로도·음주 모니터링 시스템 필요 |
| 4 | 발생해역 | ~0.12 | **남해·서해에 위험 집중** → 해당 해역 순찰 강화 |
| 5 | 연도 | ~0.04 | 연도별 추세 미미 → 구조적 문제임을 시사 |

#### 모델이 알려주는 사실

```
[이진 분류] 사상자 발생 예측
- 화재 + 어선 조합 → 사상자 발생 확률 65% 이상
- 기관손상 단독 사고 → 사상자 확률 10% 미만
- 남해 + 침몰 + 어선 → 최고 위험 패턴

[다중 분류] 위험등급 예측
- GREEN(무사) / YELLOW(경상) / ORANGE(중상) / RED(사망)
- 모델이 RED 등급을 조기 예측 → "이 조건이면 즉시 회항 권고" 자동화 가능
```

#### 정책 시사점
1. **어선 중심 안전 투자:** 전체 사고의 48%가 어선에서 발생. 어선에 이 AI 시스템을 우선 보급해야 경제적 효율이 최대화됨.
2. **예방형 대응:** 출항 전 기상 조건 + 선원 피로도를 ML 모델에 입력 → 사고 위험 예측으로 출항 통제에 활용 가능.
3. **보험료 차등화 근거:** 위험등급 예측 모델을 해상보험사에 제공하면 리스크 기반 보험료 산정 가능.

### 1.2 딥러닝 (피부 병변 분류) - 핵심 인사이트

#### Confusion Matrix가 말해주는 것

```
주요 오분류 패턴 (예상):
  mel(흑색종) → nv(점)으로 오분류 多
  → 이유: 외형이 유사하고 nv 데이터가 67%로 압도적 다수
  → 해결: Class Weight 적용 + 흑색종 데이터 증강 필수

  bcc → akiec 오분류
  → 이유: 초기 기저세포암과 광선각화증 외형 유사
  → 해결: 고해상도 촬영 가이드 + 의사 검토 권고
```

#### Per-Class F1 Score의 임상적 의미

| 클래스 | F1 목표 | 비고 |
|--------|---------|------|
| mel(흑색종) | **≥ 0.75** | F1보다 **Recall(재현율) 극대화** 우선 — 놓치면 생명 위험 |
| bcc(기저세포암) | ≥ 0.70 | 조기 발견 시 국소치료로 완치 가능 |
| nv(점) | ≥ 0.90 | 데이터 많아 자연스럽게 높음 |
| akiec(광선각화증) | ≥ 0.65 | 악성 전환 가능성이 있어 놓치면 안 됨 |

> **핵심:** 이 시스템은 "틀리면 어떤 방향으로 틀려야 하는가"를 설계해야 함.
> 양성을 악성으로 잘못 판정(False Positive)하는 것이,
> 악성을 양성으로 잘못 판정(False Negative)하는 것보다 훨씬 낫다.

#### Grad-CAM 시각화가 말해주는 것

- AI가 실제로 **병변의 경계부와 색상 불균일 영역**을 보고 판단하는지 확인 가능
- 만약 AI가 **주변 정상 피부나 배경**을 주목하면 → 데이터 편향 존재
- 발표 시 "AI가 의사처럼 병변을 본다"는 신뢰성 근거로 활용

### 1.3 딥러닝 (외상 분류) - 핵심 인사이트

- 화상(Burns)과 열상(Lacerations)의 **정확한 분류**가 핵심
  → 화상은 냉각, 열상은 압박지혈로 초기 처치가 완전히 다름
- 타박상과 열상 오분류 시 → 내출혈을 놓칠 수 있음
- **MobileNetV3 선택 이유:** 선박 엣지 디바이스(라즈베리파이 등) 탑재 가능, 추론 속도 < 200ms

## 2. 세 가지 AI의 통합 효과

```
[입력] 선원이 스마트폰으로 사진 촬영
         ↓
[ML]  이미 알려진 사고 상황? → 위험등급 예측
         ↓
[DL1] 피부 병변인가? → EfficientNet-B3 판독
[DL2] 외상인가?     → MobileNetV3 분류
         ↓
[출력] 위험등급 + 병명 + 법적 근거 있는 처치 지침
         ↓
[저장] 표준의료보고서 JSON 자동 생성
```

**단순한 병명 알려주기가 아닌, "지금 당장 무엇을 해야 하는가"를 법적 근거와 함께 제공하는 것이 경쟁력**

## 3. 현재 시스템의 한계 및 부족한 점

### 3.1 데이터 부족 (가장 중요)

| 영역 | 현황 | 문제 |
|------|------|------|
| 피부 병변 | 10,015장 | nv(점) 67% 편중 → 악성 분류 성능 저하 가능 |
